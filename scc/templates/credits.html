How does this work?

The No. 5 Crossbar (5XB) switching system originally used custom punched cards to output "Trouble Records" for technicians to use when diagnosing faults and repairing the system. These cards were punched by a KS-13834 Perforator. In the early 1980s, near the end of the machine's life, computer-aided administration was added to the 5XB by way of the Maintenance Data Transmitter (MDT). The MDT basically pretended to be a perforator, but instead of punching cards, it temporarily stored the card data and sent it to a central location known as a "Switching Control Center" (SCC) for analysis.

The SCC contained a PDP-11 minicomputer running the Automatic Trouble Analysis (ATA) software, which processed the incoming trouble data and provided technicians with information on how to fix the problem. Every central office in an area could forward its data to a single SCC, which allowed for more efficient troubleshooting and maintenance of the network.

The museum's No. 5 Crossbar system is equipped with both a Trouble Recorder and an MDT. The trouble recorder is still functional and can punch cards, but the MDT does not boot. While we could reverse engineer it, we thought it would be better to recreate its functionality using modern hardware and software. This project is the result of that effort.

The new MDT is built using an ESP32 microcontroller, which runs the code seen here. It reads the cards as they come in from the switch and sends the data to this application which (like the original SCC) processes the data, punches a virtual card, and analyzes the card data to bin the cards.

Endpoints are provided to pull the most recent card data as JSON, as well as to permalink to an arbitrary card by its ID.
